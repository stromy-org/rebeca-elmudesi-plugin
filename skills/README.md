# Skills in Rebeca Elmudesi Asset Editor

This directory holds the plugin's skills. Two kinds of skills live here:

1. **MCP-hosted stubs** — ~25-line SKILL.md files that fetch their canonical
   body from an MCP server via `ReadMcpResourceTool`. **Do not author these
   by hand.** They are generated by stromy-org's
   `scripts/sync-mcp-skill-stubs.py` driven by `sync-manifest.json`. To wire
   a new MCP-hosted skill into this plugin, add an entry to that manifest
   listing `"rebeca-elmudesi"` under the `plugins` array.

2. **Local skills** — fully authored SKILL.md files unique to this plugin.
   Frontmatter must include `_local: true` so the plugin-completeness
   validator (`scripts/validate-plugin-completeness.py` in stromy-org)
   recognises them as deliberate, not orphaned.

This plugin ships **no maintainer skill** — maintaining it (add / refresh /
release) is an operator task, not a client-facing one. `plugin-maintain`
deliberately lives only in `stromy-org/.claude/skills/plugin-maintain/` and is
never distributed in this plugin's `skills/` surface (a distributed maintainer
skill would surface as a client-visible `/rebeca-elmudesi:plugin-maintain`
command). Invoke `/plugin-maintain` from stromy-org to maintain this plugin.

## Refreshing skills

From stromy-org root:

```bash
./scripts/sync.sh mcp-skill-stubs   # materialises stubs declared in sync-manifest.json
./scripts/sync.sh client-data       # refreshes companies/rebecaelmudesi/ overlay
./scripts/sync.sh validate-plugins  # confirms every invariant still holds
```
